# Legal Extractor v2.0.0 发布说明

## 🚀 核心变更：云端智能引擎升级

LegalExtractor v2.0.0 是一个里程碑式的版本，我们彻底重构了核心提取引擎，从笨重的本地 Python 方案转型为轻量、高精度的云端 AI 方案。

### ✨ 新特性

*   **接入 PaddleOCR-VL 视觉大模型**：
    *   引入百度最先进的视觉语言模型，支持对 PDF 和图片进行深度版面分析。
    *   **完美解决乱序**：彻底解决了扫描件双栏排版、印章遮挡导致的文字乱序问题。
    *   **Markdown 结构化输出**：直接生成结构化 Markdown，精准识别“诉讼请求”、“事实与理由”等章节。
*   **极致轻量化**：
    *   移除了 200MB+ 的本地 Python 环境和 OCR 库。
    *   安装包体积缩减 **90%** (macOS 版仅 ~15MB)。
    *   完全去除了对用户本地 Python 环境的依赖，开箱即用。
*   **智能解析引擎**：
    *   **自动清洗**：内置 HTML 标签过滤器，自动去除 VLM 返回的 `<div>`, `<img>` 等干扰标签。
    *   **智能分段**：优化了换行算法，导出的 Excel 单元格现在支持完美的段落换行，不再是一团文字。
    *   **落款截断**：自动识别“此致”等结束语，确保提取内容不包含无关的法院名称和日期。
*   **性能飞跃**：
    *   引入**内存缓存机制**。同一个文件只需识别一次，后续预览或导出操作**秒级响应**，无需重复消耗 API 额度。
*   **UI/UX 升级**：
    *   加载状态增加了具体的文案提示（如“正在进行文档智能解析”）。
    *   配置面板移除过时的 MCP 选项，更加简洁。

### 🛠️ 修复与优化

*   **Fix**: 修复了 Excel 导出时单元格不自动换行的问题。
*   **Fix**: 修复了 Windows 平台下路径解析的潜在 Bug。
*   **Chore**: 标准化了 CI/CD 流程，现在通过 GitHub Actions 自动构建全平台发布包。

### 📖 升级指南

v2.0.0 采用了新的配置方式，不再依赖本地模型，而是需要配置百度 AI 开放平台的 API Key。

1.  下载并安装 v2.0.0。
2.  在软件根目录找到 `config/conf.yaml` (首次运行会自动生成)。
3.  参考 [配置指南](../user/CONFIG_GUIDE.md) 填入您的 `api_key` 和 `secret_key`。
